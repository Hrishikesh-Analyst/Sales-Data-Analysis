# Sales-Data-Analysis
📊 SQL Sales Data Analysis Project

This project demonstrates SQL data analysis on an Sales database. It includes queries for analyzing customers, orders, products, and sales data to extract meaningful business insights.

The database contains the following tables:

Customers – Stores customer details such as name, email, and city.

Products – Contains product information like name, category, and price.

Orders – Records customer orders with order date and status.

Order_Items – Includes product quantities, discounts, and total sales amount for each order.

🔍 Key SQL Operations Performed

Data filtering using WHERE conditions

Sorting results using ORDER BY

Aggregation using SUM, AVG, COUNT, MAX, MIN

GROUP BY analysis for categories and sales

JOIN operations to combine multiple tables

CASE statements for status transformation

COALESCE to handle missing values

HAVING clause for advanced filtering

📈 Insights Generated

The queries help analyze:

Customers from specific cities (e.g., Pune)

Top expensive products

Completed orders and order statuses

Total sales by customer

Revenue generated by product categories

City-wise sales performance

High performing products based on sales

Customers with multiple orders

Customers who never placed orders

🎯 Purpose of the Project

This project is designed to demonstrate practical SQL skills for data analysis, including database querying, sales analytics, and business intelligence reporting.
